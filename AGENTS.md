# AGENTS

## 커밋 규칙

- 커밋은 의미 단위로 나누어 남긴다.
- 커밋 메시지는 Conventional Commits 스타일을 따른다.
- 타입 prefix를 사용한다. 예: `feat:`, `fix:`, `docs:`, `refactor:`, `test:`, `chore:`.
- 커밋 메시지의 제목과 본문은 한국어로 작성한다.
- 커밋 메시지는 가능하면 제목 1줄로 끝내지 말고 본문도 함께 작성한다.
- 커밋 메시지 본문에는 최소한 다음을 짧게 남긴다: 무엇을 바꿨는지, 왜 바꿨는지, 어떻게 검증했는지.
- 서로 다른 성격의 변경이 함께 들어가면 본문에서 항목을 나눠 설명한다.
